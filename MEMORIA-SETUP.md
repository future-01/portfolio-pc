# Memoria — 管理者向けセットアップ・運用ガイド

`Memoria.html` をブラウザ上から編集し、家族向けにインターネット公開するための手順です。

---

## 1. 全体像

```
[管理者(みらい)]                   [家族のスマホ/PC]
    ↓                                  ↓
PCブラウザで編集 ─→ git push ─→ GitHub Pages で配信
    ↓
写真は Cloudinary にアップロード(URLだけHTMLに保存)
```

- **編集**: PC ブラウザで `Memoria.html` を開く → 管理者ログイン → 文言・アルバム・写真を編集
- **写真ストレージ**: Cloudinary 無料プラン (25GB)
- **配信**: GitHub Pages (HTML 配信、無料、容量1GB だが写真本体は Cloudinary なので余裕)
- **コスト**: ¥0

---

## 2. 初回セットアップ (1回だけ)

### 2-1. Cloudinary アカウント作成

1. <https://cloudinary.com/users/register_free> から無料登録
2. ダッシュボードで **Cloud Name** をメモ (例: `my-memoria`)
3. 左メニュー **Settings → Upload → Upload presets** に進む
4. **Add upload preset** をクリック
5. 設定:
   - **Signing Mode**: `Unsigned`
   - **Preset name**: 任意 (例: `memoria-unsigned`) — メモする
   - **Folder** (任意): `memoria/` を入れると整理しやすい
6. **Save** で保存

### 2-2. GitHub Pages 設定

1. このリポジトリを GitHub に push
2. リポジトリの **Settings → Pages** へ
3. **Source** を `Deploy from a branch` に設定
4. **Branch**: `main` / `/ (root)` を選択 → Save
5. 数分待つと <code>https://&lt;ユーザー名&gt;.github.io/&lt;リポジトリ名&gt;/Memoria.html</code> で公開される

### 2-3. サイトに Cloudinary 設定を入力 + 必須のパスワードローテート

> ⚠️ **このセクションは必ず家族に URL を共有する前に行ってください。** 初期合言葉と初期管理者パスワードは開発時のサンプルなので、必ず強い独自の値に変更してください。

1. 上記公開 URL を PC ブラウザで開く
2. 初期合言葉を入力 → 開く (初期値はみらいさんの手元メモを参照)
3. 設定タブ → **管理者でログイン** に初期管理者パスワードを入力
4. 管理者メニュー → **Cloudinary 設定** に Cloud Name と Upload preset を入れる
5. **管理者パスワードの変更** で強いパスワードへ変更 (推奨: 英大文字・小文字・数字・記号を含む 8 文字以上、辞書に載っていない合成語)
6. **合言葉の変更** で家族の合言葉も同様に変更 (初期合言葉から連想できない別の語にする)
7. 設定 → **公開・書き出し → 公開用 HTML をダウンロード** → ローカルの `Memoria.html` を上書き → `git commit && git push`
8. push 後の公開 URL を家族に共有

## 3. 日々の運用フロー

### 文言/タイトル変更

1. PC で公開 URL を開いて合言葉でログイン
2. プロフィール → 管理者でログイン
3. 編集したい文字の右に出る ✏️ をタップ → 編集 → ✓ で保存
4. 設定 → 管理者メニュー → **公開・書き出し → 公開用 HTML をダウンロード**
5. ダウンロードされた `Memoria.html` をローカルリポジトリの同名ファイルに上書き
6. `git add Memoria.html && git commit -m "update: 文言修正" && git push`
7. 数分後、家族端末でリロードすると反映される

### 写真追加

1. アルバム詳細を開く → 右下の **+** ボタン
2. 写真ファイルを選択 (複数可) → アップロード
3. Cloudinary に転送され、URL が config に追記される
4. 設定 → **公開・書き出し → 公開用 HTML をダウンロード** → 上書き → push

### アルバム追加

1. ホーム or アルバム一覧画面の右下 **+** ボタン
2. タイトル・サブタイトル・日付・説明を入力 → 作成
3. アルバム詳細に入って **+** から最初の写真を追加 (1枚目が自動的に表紙になる)
4. 表紙を変えたい場合: 写真を長押し → 表紙にする
5. 公開用 HTML をダウンロード → push

### アルバム/写真の削除

- アルバムカード or 写真サムネを **長押し** → メニューから削除
- 確認モーダルで削除実行
- 公開用 HTML をダウンロード → push
- ⚠️ Cloudinary 上のファイル本体は別途 Cloudinary 管理画面から削除する必要があります (容量管理のため)

## 4. データの仕組み

### 編集の保存先

| 種別 | 保存場所 | 永続性 |
|---|---|---|
| 編集中の設定 | ブラウザ `localStorage` (`MEMORIA_USER_DATA`) | ブラウザに残る |
| 管理者ログイン状態 | ブラウザ `localStorage` (`MEMORIA_ADMIN`) | ログアウトまで |
| 家族の合言葉認証 | ブラウザ `sessionStorage` (`MEMORIA_UNLOCKED`) | タブを閉じると消える |
| 写真本体 | Cloudinary | 永続 |
| 公開後の設定 | `Memoria.html` 内 `DEFAULT_MEMORIA_CONFIG` | git に永続 |

### 公開用 HTML 書き出しの仕組み

1. ブラウザが現在の HTML 自身を `fetch` で取得
2. `// MEMORIA_CONFIG_BEGIN` 〜 `// MEMORIA_CONFIG_END` の間を、編集後の設定 JSON に置き換える
3. 結果を Blob として download

つまり:
- 編集 = `localStorage` に保存
- 公開 = HTML ファイル本体に焼き込み
- push 後は `localStorage` がなくても初期値として正しく表示される

### 設定の入出力 (バックアップ)

設定 → 公開・書き出し から:
- **エクスポート**: 現在の設定を `memoria-config.json` で出力 (PCに保存しておくとバックアップになる)
- **インポート**: JSON ファイルから復元
- **初期状態に戻す**: `localStorage` をクリア (HTML 本体は変わらない)

## 5. トラブルシューティング

| 症状 | 対処 |
|---|---|
| 公開用 HTML ダウンロードボタンを押すとエラー | `file://` 直開きでは動きません。公開 URL か Live Server 経由で開いてください |
| 写真アップロードが失敗する | Cloudinary 設定を再確認。Upload preset が **Unsigned** モードか確認 |
| 編集が反映されない | ブラウザのリロードが必要です。Ctrl+Shift+R で強制リロード |
| 別の PC で編集を続けたい | エクスポートした JSON を新しい PC で読み込む or git push 後にその PC でアクセスして再編集 |
| 家族が古い表示のまま | GitHub Pages の反映に数分かかります。ブラウザキャッシュも疑ってください |
| 合言葉/管理者パスワードを忘れた | パスワードはハッシュ化されているため平文編集はできません。最初のクリーンコミット (`feat: add admin customization with PBKDF2-hashed authentication`) の `Memoria.html` から `auth` ブロックの初期ハッシュを `git checkout <SHA> -- Memoria.html` で取り戻し、サイト上で再ログイン後すぐに新しい合言葉へローテートしてください |

## 6. セキュリティ上の注意

### 認証の仕組み

合言葉と管理者パスワードは **PBKDF2-SHA256 (310,000 反復)** でハッシュ化されたうえで `Memoria.html` に保存されています。GitHub のリポジトリを直接見ても、合言葉そのものは読み取れません。

ただし以下の限界があります。

- **弱い合言葉は辞書攻撃で復元可能**: `password` / `12345678` のような既知単語や、家族・地名・誕生日などすぐ思いつく語はハッシュ化していても破られます。長く・規則性のない合言葉を選んでください。
- **JavaScript の bypass は防げない**: クライアント側だけの認証なので、ブラウザの DevTools に詳しい人なら突破できます。
- **本当に機密性の高い写真は載せないでください**。家族向け簡易ロックと割り切る前提です。
- 過去コミット履歴に **古い合言葉のハッシュ**が残ることがあります。これも辞書攻撃の対象になり得るので、**現行の合言葉は古いものと連想できない別の語**を選んでください。

### Cloudinary

- Upload preset は Unsigned のため、プリセット名が漏れると誰でもアップ可能になります。Cloudinary 側でファイルサイズ上限・フォルダ制限を設定して被害を限定するのを推奨します。
- 削除した写真は Cloudinary 側に残ります。容量管理のため定期的に Media Library から物理削除してください。

### リポジトリ公開設定

- リポジトリは **Public でも Private でも動きます**。
- Public の場合、ハッシュ化済みでも上記の通り辞書攻撃リスクがあるため、強いパスワードと写真の選別が必須です。
- 内容が一切見られたくない場合は Private + GitHub Pages も Private 配信 (有料) を検討してください。

## 7. 合言葉を変更したくなったとき

家族構成が変わった、合言葉が漏れた可能性がある、定期ローテートしたい、などのときの手順:

1. PC ブラウザで公開 URL を開いて現行の合言葉でログイン
2. プロフィール → 管理者でログイン (現行の管理者パスワード)
3. 管理者メニュー → **管理者パスワードの変更** または **合言葉の変更**
   - いずれも **強いパスワード** を選ぶ (8 文字以上、辞書語/家族の名前/誕生日などを避ける)
4. 設定 → **公開・書き出し → 公開用 HTML をダウンロード**
5. ダウンロードされた `Memoria.html` をローカルリポジトリに上書き
6. `git add Memoria.html && git commit -m "chore: rotate credentials" && git push`
7. **新しい合言葉を家族に共有**。古い合言葉は破棄。

### 古いブラウザでログインできない場合

ローテート後、家族が以前から開いていたタブ/ブラウザでは、`localStorage` に古いハッシュがキャッシュされていることがあります。Ctrl+Shift+R で強制リロードしてもらってください。それでもおかしければ、設定 → **初期状態に戻す** で localStorage をクリアし、新合言葉で再開錠できます。
